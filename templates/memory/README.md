# Memory layout (Anthropic Memory Tool-compatible)

These templates adopt the **Anthropic Memory Tool** protocol (`memory_20250818`) for consistency with Anthropic's gold standard. Copy them into `~/.claude/projects/{slug}/memory/` at `/assistant init` time.

## Files

- **`progress.md`** — bootstrap required. "What's done / next / open questions". Read at every session start. Updated at every session end.
- **`feature-checklist.md`** — frozen scope. "In / out / acceptance / non-negotiables". Prevents scope drift.
- **`reflection.yaml`** — Reflexion pattern. One per significant session. Fed to the memory pattern scan for cross-project lesson extraction.

## Auto-memory files (written by Claude Code itself)

- `MEMORY.md` — index (already generated by CC).
- `{user,feedback,project,reference}_*.md` — atomic facts with YAML frontmatter. Claude asks to save these during sessions.

## Indexing

Run `bun scripts/index-memories.ts` to refresh the SQLite FTS5 index at `~/.claude/assistant-index.db`. Search with `bun scripts/recall.ts "your query"`.

## Why this layout

- **progress.md/feature-checklist.md**: matches Anthropic's official cookbook. Future-proof when CC adopts the Memory Tool directly.
- **reflection.yaml**: enables the Reflexion loop — the memory pattern scan identifies patterns like "user keeps correcting me about X" and surfaces fixes into `references/`.
- **SQLite index**: query 500+ markdown files in <10ms, zero npm deps.
