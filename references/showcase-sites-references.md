## Feeds to Follow

Sources for ongoing inspiration and keeping the catalog fresh.

### Awwwards
**URL** — `awwwards.com`
Daily Site of the Day, monthly/yearly awards. Highest signal-to-noise for motion-heavy sites. Check weekly.

### FWA (Favourite Website Awards)
**URL** — `thefwa.com`
Oldest web awards (2000+). Tends heavier/experimental than Awwwards. Historical reference for 3D/WebGL work.

### Godly
**URL** — `godly.website`
Curation-only site, visually dense grid. Good for mood-board building — quick scan produces direction.

### SiteInspire
**URL** — `siteinspire.com`
Minimalist curation, emphasis on craft over spectacle. Counter-balance to Awwwards when you need restraint.

### One Page Love
**URL** — `onepagelove.com`
Curates single-page sites specifically. Useful for showcase sites with no deep navigation.

### Httpster
**URL** — `httpster.net`
Eclectic, unpredictable. Good for breaking out of the Awwwards filter bubble.

### Codrops
**URL** — `tympanus.net/codrops`
Tutorials, not just curation. Source for learning the actual techniques behind what's on Awwwards (shader patterns, scroll effects, GSAP sequences).

### Page Collective
**URL** — `pagecollective.com`
Curated single-page showcase. Mobile-first curation, useful lens.

## Usage in `/site` Moodboard

- Pick **3-5 reference sites** per project during the moodboard phase.
- Tag each with 3-5 descriptors.
- Build MOODBOARD.md with URL + screenshot + 1-line reason for each.
- Cross-check all references share ≥1 tag — if they don't, the moodboard is incoherent and the design direction needs a second pass with the client.
